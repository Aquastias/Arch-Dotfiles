# Installer `--unattended` flag

Status: ready-for-agent
Category: enhancement

## Parent

`.scratch/vm-test-harness/PRD.md`

## What to build

Add a permanent unattended-install mode to the installer entry point. When passed `-y` or `--unattended`, every interactive prompt that currently blocks the install must be skipped with the safe default — same outcome as a human typing the affirmative answer at every prompt.

Three prompts are in scope:

- The disk-exclude prompt in the wipe step (current default: Enter = no exclusions). Skip → no exclusions.
- The "type WIPE to confirm" prompt in the wipe step. Skip → treat as confirmed.
- The "Proceed with installation?" final summary prompt invoked by `print_summary`. Skip → treat as confirmed.

The mechanism: parse the flag in the single entry point, export an environment variable (e.g., `INSTALL_UNATTENDED=1`) that the called scripts and `confirm` helper consult, and forward the flag explicitly to the numbered scripts so they remain individually runnable in unattended mode. The hostname-prompt path stays unchanged — callers fill `install.jsonc` themselves before invoking.

The non-Arch-ISO confirmation in the bootstrap script is out of scope: it only fires when the live environment looks wrong, which the test harness will never trigger.

The encryption passphrase collection is out of scope: it only runs when `options.encryption` is true; the test harness keeps it false.

## Acceptance criteria

- [ ] Single entry point accepts `-y` / `--unattended` and propagates it to the numbered scripts
- [ ] Numbered scripts also accept the flag directly (so each remains individually runnable in unattended mode)
- [ ] In unattended mode, the wipe-step disk-exclude prompt is bypassed with the same outcome as pressing Enter
- [ ] In unattended mode, the wipe-step "WIPE" confirmation is bypassed and the wipe proceeds
- [ ] In unattended mode, the final `print_summary` "Proceed?" confirmation is bypassed and the install proceeds
- [ ] In interactive mode (no flag), all three prompts behave exactly as today — no regression
- [ ] `--help` (or equivalent) documents the flag
- [ ] Shellcheck passes on every modified script
