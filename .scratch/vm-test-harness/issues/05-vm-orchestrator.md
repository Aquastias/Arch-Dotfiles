# VM test harness orchestrator

Status: ready-for-agent
Category: enhancement

## Parent

`.scratch/vm-test-harness/PRD.md`

## What to build

The top-level test harness script that composes the modules from issues 01–04 into a single user-facing command: download the latest Arch ISO if needed, create or reuse a libvirt VM, run the installer inside the live CD, and capture the installer's logs to a file in `.os/`.

The script lives at `.os/setup-vm-for-testing.sh` and is the only user-invoked surface in this PRD.

### Configuration

All tunables exposed as variables at the top of the script. The repo URL must be one such variable. Other constants: VM name, RAM, vCPU count, disk size, ISO download directory, cache directory for the seed ISO, log file path, timeout (override via env var `VM_TEST_TIMEOUT`), test hostname.

### CLI surface

- `--recreate` — force-destroy and undefine the existing VM (including its storage and NVRAM) before creating a fresh one.
- `--help` — usage text.

No other flags. Tunables are edited at the top of the script.

### Flow

1. **Dependency check.** If `virt-install`, `virsh`, or `cloud-localds` are missing, install them via `sudo pacman -S --needed` (the package names: `virt-install`, `libvirt`, `cloud-image-utils`). If the user is not in the `libvirt` group, fail with a clear instruction to run `usermod -aG libvirt $USER` and re-login. If `libvirtd` is inactive, `sudo systemctl enable --now libvirtd`.

2. **Resolve ISO.** Use the ISO resolver module (issue 02) against the configured downloads directory.

3. **Generate seed.** Use the seed generator module (issue 03), parameterised with the configured repo URL and test hostname, output to the cache directory. Regenerated every run.

4. **VM lifecycle.**
   - If `--recreate`, destroy and undefine the existing VM (with storage and NVRAM removal) before proceeding.
   - If the named domain does not exist, create it with `virt-install` using: UEFI firmware, configured RAM/vCPU, a SATA-bus qcow2 disk of the configured size, the resolved ISO as a SATA CD-ROM (read-only), the seed ISO as a second SATA CD-ROM (read-only), default libvirt NAT network, serial pty console, no autoconsole, no autoreboot, boot order CD-ROM then HD.
   - If the domain exists and is running, force it off, then start it. If it exists and is off, start it.

5. **Log capture.** Truncate the configured log file. Run `virsh console --force` against the VM in the background, piped through `tee` to the log file.

6. **Wait.** Use the sentinel watcher module (issue 04) against the log file with the configured timeout. Receive an exit code.

7. **Cleanup.** Stop the background console capture. Print the absolute path of the log file. Exit with the code returned by the sentinel watcher (124 on timeout).

### Gitignore

Add `.os/.gitignore` containing `.vm-test/` and `testing-vm-logs`.

## Acceptance criteria

- [ ] `.os/setup-vm-for-testing.sh` exists, is executable, and exposes the configuration variables described above at its top — including a single `REPO_URL` variable
- [ ] `--help` prints usage and lists the two flags (`--recreate`, `--help`)
- [ ] First run on a clean host: dependency check installs missing packages via pacman, then proceeds
- [ ] User-not-in-libvirt-group case fails fast with a usermod hint instead of attempting libvirt operations
- [ ] First-run with no cached ISO downloads it; subsequent runs reuse the cached ISO without re-downloading
- [ ] First-run with no existing VM creates one via `virt-install` matching the spec from the PRD
- [ ] Subsequent runs with the existing VM force it off and start it again, producing a fresh log
- [ ] `--recreate` undefines the VM including its storage and NVRAM and re-creates it cleanly
- [ ] Log file is overwritten (not appended) on every run
- [ ] The script's exit code is the installer's exit code on completion, or 124 on timeout
- [ ] Background console capture is killed before the script exits, in both success and timeout paths
- [ ] `.os/.gitignore` (or the appropriate root gitignore) excludes `.vm-test/` and `testing-vm-logs`
- [ ] Shellcheck passes
- [ ] The script is verified by running it end-to-end on the user's host: VM is created, ISO is downloaded once, installer runs to completion, log file contains installer output ending with the sentinel line, exit code is correct
