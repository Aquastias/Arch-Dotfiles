# ISO resolver module

Status: done
Category: enhancement

## Parent

`.scratch/vm-test-harness/PRD.md`

## What to build

A deep, testable module that, given a downloads directory, returns the path to a usable Arch ISO — downloading the latest one only if it is not already present.

Public interface: one function that takes a downloads directory path and prints the absolute path of a usable ISO file in that directory on stdout. Behavior:

- Resolve the canonical filename of the current Arch ISO by issuing an HTTP HEAD against `https://archlinux.org/iso/latest/archlinux-x86_64.iso` and following the redirect to get the final filename (`archlinux-YYYY.MM.DD-x86_64.iso`).
- If a file with that filename already exists in the downloads directory, return its path. No network download.
- Otherwise, download via the latest URL into the downloads directory and return the new file's path.
- No checksum verification.
- The downloads directory must already exist; it is not created by this module.

The module must have no libvirt dependency and must be sourceable in isolation. The HTTP and download steps must be invoked through helper commands that can be replaced/stubbed in tests (e.g., a thin internal seam).

## Acceptance criteria

- [ ] A new module under `.os/lib/` exposes a single public function whose contract is "given a downloads dir, print path to usable ISO"
- [ ] When the resolved-latest ISO is already in the downloads dir, the function does not invoke the downloader
- [ ] When the resolved-latest ISO is not present, it downloads via the latest URL and returns the new path
- [ ] Network failures (HEAD or download) surface as non-zero exit, not a silent fallback
- [ ] No checksum verification logic is added
- [ ] Bats tests under `.os/tests/` cover: cached-hit (no download), cache-miss (download invoked), and HEAD failure (non-zero exit). Tests stub `curl` or whatever HTTP helper the module uses
- [ ] All bats tests pass
- [ ] Shellcheck passes
