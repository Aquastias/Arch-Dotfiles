# Install Config schema + Environment Config Validation

Status: done
Category: enhancement

## Parent

`.scratch/environment-selection/PRD.md`

## What to build

Add a top-level `"environment"` key to `install.jsonc` with two fields: `"desktop"` (string or array, nullable) and `"gpu"` (string or array). This is an additive change — `post_install.desktop` stays in place until issue #04 replaces the extras runner.

Add an Environment Config Validation function called during `load_config()`, before any disk writes or layout planning. It checks `"desktop"` values against the allowed set (`"kde"`, `"hyprland"`) and `"gpu"` values against the allowed set (`"amd"`, `"nvidia"`, `"intel"`, `"auto"`). On any unrecognised value it aborts with a descriptive error that lists valid options.

Update the install summary (printed before the final confirmation prompt) to show the resolved desktop selection, GPU vendor(s), and whether audio will be installed.

- [ ] `"gpu": ["amd", "nvidia"]` passes validation

**Category:** enhancement
**Summary:** Add `"environment"` key to Install Config and validate desktop/GPU values before disk writes.

**Current behavior:**
`install.jsonc` has `post_install.desktop.kde` (boolean). There is no `"environment"` key, no GPU field, and no validation of desktop or GPU values. The install summary does not mention desktop, GPU, or audio.

**Desired behavior:**
`install.jsonc` gains a top-level `"environment"` key with `"desktop"` (string or array, nullable) and `"gpu"` (string or array) fields. The config loader calls an Environment Config Validation function early — before layout planning, before disk writes — that checks each value in `"desktop"` and `"gpu"` against its allowed set and aborts with a clear error listing valid options on any unknown value. The install summary includes resolved desktop, GPU, and audio (PipeWire / none) lines before the confirmation prompt.

This is an additive change: `post_install.desktop` is not removed here (issue #04 does that). Both keys coexist temporarily.

**Key interfaces:**
- `"environment"` key in `install.jsonc` — two fields: `"desktop"` (nullable string or string array, valid values: `"kde"`, `"hyprland"`) and `"gpu"` (string or string array, valid values: `"amd"`, `"nvidia"`, `"intel"`, `"auto"`, or array of those). Documented with inline comments.
- Environment Config Validation — function called inside `load_config()`. Normalises `"desktop"` to an array (string → single-element array, null/missing → empty array). Normalises `"gpu"` similarly (excluding `"auto"`). Iterates each element; aborts non-zero with descriptive error on unknown value.
- Install summary — existing summary function gains three new lines: desktop (resolved array or "none"), GPU (resolved array or value), audio ("pipewire" or "none").
- BATS test file — new, lives alongside existing tests in `.os/tests/`. Covers all valid and invalid desktop/gpu combinations using fixture configs in a `$TEST_DIR` temp directory, matching the pattern in `configs.bats`.
