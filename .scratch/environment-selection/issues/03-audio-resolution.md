# Audio Resolution

Status: done
Category: enhancement

## Parent

`.scratch/environment-selection/PRD.md`

## What to build

Implement the Audio Resolution function in the config loading module. It derives the audio package list from the resolved desktop array — no config key is read.

If the resolved desktop array is non-empty (any desktop selected), append `pipewire pipewire-pulse pipewire-alsa wireplumber` to `packages.groups.audio` so they are pulled into pacstrap alongside everything else.

If the resolved desktop array is empty (server install — `"desktop": null` or key omitted), leave `packages.groups.audio` empty. No audio packages are installed.

Audio Resolution runs at config-load time, after the desktop value is normalised to an array and before pacstrap.

- [ ] BATS tests cover all four cases above
