# Environment Runner + KDE Desktop Environment Adapter update

Status: ready-for-agent
Category: enhancement

## Parent

`.scratch/environment-selection/PRD.md`

## What to build

Replace the explicit `post_install.desktop.kde` branch in the extras runner (`lib/chroot/extras.sh`) with a dynamic dispatch loop. The runner normalises `ENVIRONMENT_DESKTOP` (space-separated, passed from the host into the chroot) to an array, then iterates and invokes `extras/desktop/<de>/<de>.sh` by path convention. No DE name appears as a literal string in the runner. The runner continues to handle `post_install.backup` and `post_install.security` unchanged.

The full resolved desktop array is passed as `ENVIRONMENT_DESKTOP` so each adapter can inspect it for DM conflict guarding.

Update the KDE Desktop Environment Adapter (`extras/desktop/kde/kde.sh`) to read `ENVIRONMENT_DESKTOP` and skip SDDM installation only if a conflicting DM condition requires it. In practice SDDM always wins when KDE is present — the guard is defensive. No changes to KDE package selection or `install-kde.jsonc` structure.

Remove `post_install.desktop` from `install.jsonc` (now superseded by `"environment".desktop`).

## Acceptance criteria

- [ ] `ENVIRONMENT_DESKTOP="kde"` causes only the KDE adapter to be invoked
- [ ] `ENVIRONMENT_DESKTOP="hyprland"` causes only the Hyprland adapter to be invoked (even before it exists — runner emits a clear "script not found" error rather than a silent no-op)
- [ ] `ENVIRONMENT_DESKTOP="kde hyprland"` causes both adapters to be invoked in order
- [ ] `ENVIRONMENT_DESKTOP=""` (no desktop) causes no adapter to be invoked; `post_install.backup` and `post_install.security` still run if set
- [ ] No DE name (including `"kde"`) appears as a literal string in the runner loop body
- [ ] KDE adapter receives `ENVIRONMENT_DESKTOP` and uses it; SDDM is installed and enabled when `"kde"` is in the array
- [ ] `post_install.desktop` key removed from `install.jsonc`; existing `post_install.backup` and `post_install.security` keys unchanged
- [ ] BATS tests for the runner cover: single DE, dual DE, empty, unknown DE (error path)
- [ ] BATS tests for the KDE adapter cover: SDDM install when KDE-only, SDDM install when KDE+Hyprland
- [ ] Shellcheck passes on all modified scripts

## Agent Brief

**Category:** enhancement
**Summary:** Replace KDE-specific `if` branch in extras runner with dynamic DE adapter dispatch; pass desktop array to adapters.

**Current behavior:**
The extras runner (`lib/chroot/extras.sh`) has an explicit `if` branch that checks `post_install.desktop.kde` and sources `extras/desktop/kde/kde.sh` directly. Adding any new DE requires editing the runner. The KDE adapter does not receive information about which other DEs are selected.

**Desired behavior:**
The extras runner normalises `ENVIRONMENT_DESKTOP` (space-separated env var, passed from host into chroot) to an array. It iterates the array and sources `extras/desktop/<de>/<de>.sh` by path convention — no DE name is a literal string in the runner. An unrecognised DE (script not found) produces a clear error and non-zero exit rather than a silent skip.

`ENVIRONMENT_DESKTOP` is exported before each adapter is sourced so adapters can inspect the full selection (used for DM conflict guarding).

The KDE adapter (`extras/desktop/kde/kde.sh`) is updated to read `ENVIRONMENT_DESKTOP`. The SDDM install path is unchanged — SDDM is always installed when KDE is present. The guard is defensive: the adapter checks that it is not being asked to install a conflicting DM when one is already handled by a sibling adapter. In practice this is a no-op for SDDM today.

`post_install.desktop` is removed from `install.jsonc`. The runner's `post_install.backup` and `post_install.security` branches are unchanged.

**Key interfaces:**
- Environment Runner loop — iterates `ENVIRONMENT_DESKTOP` array; invokes `extras/desktop/${de}/${de}.sh`. No adapter name is hardcoded. Exports `ENVIRONMENT_DESKTOP` before each invocation.
- `ENVIRONMENT_DESKTOP` env var — space-separated string, set by the host-side config loader from the resolved desktop array before arch-chroot. Adapters read it to inspect co-selected DEs.
- KDE adapter — reads `ENVIRONMENT_DESKTOP`; SDDM install and enable unchanged.
- `post_install.desktop` — removed from `install.jsonc` template and from the runner.
- BATS tests — stub adapter scripts as executable test doubles that write to a sentinel file when called. Cover: single DE, dual DE, empty, unknown DE (error). KDE adapter tests: mock `pacman` and `systemctl`; assert SDDM installed in both kde-only and kde+hyprland cases.

**Acceptance criteria:**
- [ ] `ENVIRONMENT_DESKTOP="kde"` → only KDE adapter invoked; `backup`/`security` extras unaffected
- [ ] `ENVIRONMENT_DESKTOP="kde hyprland"` → both adapters invoked in order
- [ ] `ENVIRONMENT_DESKTOP=""` → no adapter invoked
- [ ] Unknown DE name → runner exits non-zero with "script not found" style error
- [ ] No DE name literal in runner loop body
- [ ] KDE adapter: SDDM installed and enabled when `"kde"` in `ENVIRONMENT_DESKTOP`
- [ ] `post_install.desktop` removed from `install.jsonc`; backup/security keys untouched
- [ ] BATS tests pass for all runner cases and KDE DM guard
- [ ] Shellcheck passes
