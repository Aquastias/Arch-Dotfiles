# Audit: assert no _fixture/ under .os/programs/

## Parent

`.scratch/config-generator-finalization/PRD.md`

## What to build

After slice 04 relocated `programs/_fixture/hello/` to
`.os/tests/fixtures/programs/hello/`, the absence of any
`_fixture/` under `.os/programs/` is a convention — not enforced.
A future contributor (or sloppy revert) could re-introduce a
production fixture program without any test catching it.

Add a single check in `tests/audit.sh` that fails loudly if any
`_fixture/` directory exists under `.os/programs/`. The check
costs ~one find invocation and locks in the ADR 0013 +
slice-04 contract that production trees ship no fixture.

- [ ] Check passes on `main` as-is (post slice 04)

## Agent Brief

**Category:** enhancement
**Summary:** Lock in the ADR 0013 + slice-04 contract — production
trees ship no fixture program — by adding an audit check.

**Current behavior:**
After slice 04, `.os/programs/_fixture/` no longer exists. Nothing
in the test suite asserts this property. A future contributor (or
a sloppy revert) could re-introduce a `_fixture/` directory under
`.os/programs/` and every existing check would still pass.

**Desired behavior:**
`tests/audit.sh` runs a new check that searches `.os/programs/`
for any directory named `_fixture` (at any depth) and fails loudly
when one is found. The check sits alongside the existing
`programs/` traversals as its own numbered section, using the same
`_pass` / `_fail` helpers as every other check in the file.

**Key interfaces:**
- `audit.sh` section format — `── N. <title> ──` header, then one
  or more `_pass` / `_fail` calls per finding. Existing sections
  iterate `${OS}/programs` via `find`; follow that pattern.
- The check must be self-describing: the failure message names the
  offending path so the contributor sees exactly what to remove.

**Acceptance criteria:**
- [ ] `tests/audit.sh` fails when any `_fixture` directory exists
      under `.os/programs/` (verify by creating one in a scratch
      worktree and re-running)
- [ ] `tests/audit.sh` passes on current `main` (post slice 04)
- [ ] Check uses the same `_pass`/`_fail` helpers and section
      header style as the existing checks
- [ ] One commit, scoped to `tests/audit.sh` only

**Out of scope:**
- Renaming or restructuring existing audit sections
- Asserting on `.os/tests/fixtures/programs/` shape (separate
  concern; that tree is consumed by bats, not by the installer)
- A pre-commit hook or CI wiring (audit.sh is already wired into
  `tests/run.sh`)
