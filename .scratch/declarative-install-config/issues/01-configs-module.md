# Config loader and merger module

Status: ready-for-agent
Category: enhancement

## Parent

`.scratch/declarative-install-config/PRD.md`

## What to build

Build `.os/lib/configs.sh` — the deep, testable module that loads a host or user config by name and returns the merged result of core + specific. The public interface is two functions: load-and-merge a host config given a hostname, load-and-merge a user config given a username. The module encapsulates merge semantics: list fields concatenate (with dedupe), scalar fields in the specific config override core, missing fields fall back to whichever side has them. No side effects — no chroot calls, no filesystem mutation beyond reading config files. Pure enough to be tested in isolation.

Set up bats test infrastructure under `.os/tests/` and write unit tests covering all the merge cases.

## Acceptance criteria

- [x] `.os/lib/configs.sh` exists with two public functions: load_host_config, load_user_config
- [x] List concatenation with dedupe is implemented for `users`, `system_programs`, `programs`, `groups`
- [x] Scalar override is implemented for `shell`, `sudo`
- [x] Missing core file produces a hard error
- [x] Missing specific config (host or user) is communicated via return value, not a hard error (so callers can handle graceful degradation)
- [x] `.os/tests/configs.bats` covers: core+empty specific, empty core+specific, list concatenation with dedupe, scalar override, missing field on either side, missing core, missing specific
- [x] All bats tests pass
- [x] `core` is treated as a reserved name (rejecting an attempt to load `hosts/core` or `users/core` as a real entity)
