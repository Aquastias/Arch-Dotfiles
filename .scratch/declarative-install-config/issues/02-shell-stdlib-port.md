# Port shell-stdlib into .os/lib/

Status: done
Category: enhancement

## Parent

`.scratch/declarative-install-config/PRD.md`

## What to build

Port `.pkglist/shell-commons/shell-stdlib.sh` into `.os/lib/shell-stdlib.sh`. The new file becomes the canonical stdlib for all program install scripts inside `.os/programs/`. The original under `.pkglist/` is left untouched (reference material only).

The port should preserve the existing utility surface: string helpers, output helpers (print_status), array helpers, command-existence checks, root checks, directory checks, desktop-environment detection, package-installed checks, notification helpers.

- [x] `.os/lib/shell-stdlib.sh` exists and is sourceable
- [x] All public functions from `.pkglist/shell-commons/shell-stdlib.sh` are present with the same names and signatures
- [x] Sourcing the new stdlib in a test script and calling each public function succeeds
- [x] `.pkglist/shell-commons/shell-stdlib.sh` is unchanged
