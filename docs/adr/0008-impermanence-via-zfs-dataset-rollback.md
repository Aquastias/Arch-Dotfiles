# Impermanence via ZFS dataset rollback

## Status

Accepted.

## Context

Opt-in "impermanence" feature: declared paths persist, everything else resets on reboot. NixOS achieves this cleanly because `/` is mostly symlinks into `/nix/store/`, so rolling back `/` does not erase the system itself. Arch has no equivalent indirection — `/usr` *is* the system — so naively rolling back `rpool/ROOT/arch` on every boot would undo every pacman update.

## Decision

Implement Impermanence via per-dataset ZFS rollback to a `@blank` snapshot taken once at install time, scoped to finer-grained datasets than `/`:

- **Rolled back:** `rpool/ROOT/etc`, `rpool/ROOT/root`, `rpool/ROOT/opt`, `rpool/ROOT/srv`, `rpool/ROOT/usrlocal`.
- **Not rolled back:** `rpool/ROOT/arch` (i.e. `/`, `/usr`, `/boot` host-side), the existing `/home`, `/var`, `/var/log`, `/var/cache`.

Persistent state lives on a new `rpool/persist` dataset (name and mountpoint configurable via `options.impermanence.{dataset,mount}`), bind-mounted into rolled-back paths via systemd `.mount` units. Two layers of mount configuration:

- **Bootstrap** in `/usr/lib/systemd/system/` + `/usr/lib/tmpfiles.d/`: binds `/persist/etc/systemd/system` and `/persist/etc/tmpfiles.d` over their `/etc` counterparts. Lives on the non-rolled-back root dataset, immune to rollback by virtue of location.
- **Payload** in `/persist/etc/systemd/system/`: per-path `.mount` units, generated by the installer from Curated Persist Defaults plus host-declared `persist.directories`/`persist.files`. Operator-editable at runtime.

A mkinitcpio Rollback Hook reverts each Rollback Dataset to `@blank` in initramfs before mount; fails closed if `@blank` is missing. A pacman post-transaction hook re-snapshots `@blank` on every Rollback Dataset so package-shipped `/etc/<pkg>/` files survive across reboots.

The feature is install-time only: toggling Impermanence later requires a re-install.

## Considered alternatives

- **Roll back `rpool/ROOT/arch` itself, re-snapshot `@blank` on every pacman transaction.** Mirrors NixOS's "reset all of /" surface promise, but the rollback target becomes "state at last pacman run" rather than "state at install," weakening the guarantee. Rejected.
- **tmpfs root + selective persistent mounts.** Requires reworking the boot stack and doesn't compose with ZFS-on-root. Too large a deviation.
- **Overlayfs root (RO lower + tmpfs upper).** Updates require re-imaging. Doesn't fit Arch's rolling-release model.
- **Strict pre-pacman wipe (`zfs rollback` before every transaction).** Closes the small "edits-before-pacman survive one reboot" leak but silently destroys uncommitted `/etc/*` work the moment the operator runs pacman. Rejected for v1 as too footgun-y; revisit as opt-in drift-check hook.
- **All persist config in `/usr/lib` (snapshot-baked).** Forces a re-snapshot dance for every persist-list change. Rejected; bootstrap-in-`/usr/lib` + payload-in-`/persist` split keeps the persist list mutable while still protecting the bootstrap.

## Consequences

- The `@blank` snapshot for each Rollback Dataset is regenerated by the Pacman Resnapshot Hook on every transaction. User edits to non-persisted paths made *before* a pacman run survive one additional reboot — a small documented leak.
- New code surface: a fresh `lib/chroot/impermanence.sh`, an initcpio hook pair, a pacman hook script, ~12 generated `.mount` units per host, validation rules in `lib/validation.sh` for the `persist` field.
- `/etc` becoming a dedicated dataset is unusual on Arch — must mount before most early-boot services. Handled by standard ZFS `mountpoint` properties + systemd's `zfs-mount-generator`.
- Operators discover the persist mechanism via `os impermanence status` (it
  lists the active Persist Mounts by filtering mount units whose `What=` is
  under the Persist Dataset). The generated bind units are named after their
  mount point per the systemd `.mount` contract (`<esc>.mount`, e.g.
  `etc-ssh.mount`), so there is no `persist-*.mount` glob to match.
- Loss of `/etc/secrets/` (the Machine Age Key) or `/etc/ssh/` (host keys it's derived from) would break the SOPS Runtime Service. Both are in Curated Persist Defaults specifically to prevent this.
